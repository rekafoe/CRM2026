# Редактор макетов и префлайт — резюме для продолжения

## Сделано

### Префлайт
- Проверка макетов (PDF, JPG, PNG, TIFF): размеры, вылеты, DPI.
- Превью в модалке с линией обрезки.

### Каталог шаблонов
- Таблица `design_templates`, CRUD, загрузка превью.
- Страница `/adminpanel/design-templates`.
- Превью шаблона отображается как **фон в редакторе** (PNG/JPG; INDD не поддерживается).

### Редактор макетов
- Konva-канвас: страница по формату шаблона (spec: width_mm, height_mm).
- Фоновое изображение из `template.preview_url`.
- **Зоны обрезки:** линия обрезки (красная пунктир) и безопасная зона 5 мм (бирюзовая). Кнопка «Зоны обрезки» вкл/выкл. В экспорт PNG не попадают.
- **Текстовые блоки:** кнопка «Добавить текст», перетаскивание, выбор и правка в тулбаре (текст и размер шрифта). Сохраняются в designState.
- **Мультистраничность:** из spec шаблона читается `page_count` (по умолчанию 1). В тулбаре при page_count > 1 — переключатель «Страница N / M» (предыдущая/следующая). Контент (фото, текст) хранится по страницам; при сохранении в designState уходит массив `pages`; в заказ экспортируется текущая страница как PNG.
- **Сайдбар и панели:** слева вертикальное меню (Фото, Текст, Шаблоны, Фон, Коллажи, Стикеры, Клипарты, Рамки). При выборе открывается правая панель. В панели «Фото»: кнопка «Добавить изображение», сортировка по названию/дате, фильтр, тумблеры «Автозаполнение» и «Скрыть используемые изображения», зона загрузки (клик или drag-and-drop). В панели «Текст»: добавление текста и правка выбранного. Остальные разделы — заглушка.
- Добавление фото, перетаскивание.
- Экспорт в PNG (текущая страница). **Экспорт в PDF:** кнопка «Экспорт в PDF» — весь макет (все страницы) в один PDF, каждая страница макета = один лист PDF, размер по spec (мм), без направляющих.
- **Мобильная оптимизация:** на экранах ≤768px при экспорте в PDF/PNG используется pixelRatio 1 (меньше памяти и быстрее). Захват кадров по двойному requestAnimationFrame вместо фиксированной задержки. Во время экспорта в PDF показывается прогресс (например «Экспорт PDF… 2/5»). В CSS — медиазапрос для мобильных: кнопки и поля не менее 44px по высоте, touch-action, удобная прокрутка канваса.

### Интеграция с заказом
- Кнопка «Создать макет» в FilesModal → каталог → редактор.
- Сохранение: загрузка PNG в заказ и обновление `params` позиции (designState, designTemplateId).

### Backend
- PATCH items поддерживает `params` (слияние с существующими).

### Префлайт: статус в списке
- В модалке файлов заказа (FilesModal) у каждого файла, поддерживающего префлайт (PDF, JPG, PNG, TIFF), рядом с именем отображается статус проверки после того, как пользователь один раз нажал «Проверить макет». Кэш результатов по fileId: ок (зелёная галочка), предупреждения (жёлтый «!»), ошибки (красный значок). Если префлайт ещё не запускали — показывается серый щит (подсказка «Нажмите щит для проверки»).

### Префлайт: авто-проверка при загрузке
- При загрузке файла в заказ (PDF, JPG, PNG, TIFF) префлайт запускается автоматически; результат попадает в кэш и сразу отображается в списке (иконка статуса без необходимости нажимать «Проверить макет»).

### Роутинг редактора в основном приложении
- В `main.tsx` добавлены маршруты `/design-templates` и `/design-editor/:templateId` (с RequireAuth). Редактор и каталог доступны как из админки (`/adminpanel/design-templates`, `/adminpanel/design-editor/:id`), так и по корневым путям. Кнопка «Создать макет» в FilesModal ведёт на `/design-templates?orderId=…`; каталог и редактор по `location.pathname` выбирают «назад» и ссылки (корневой путь или `/adminpanel`).
