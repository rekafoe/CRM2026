# ТЗ для дизайнеров: подготовка макета под импорт в CRM

Документ для CorelDRAW / Illustrator. Цель — чтобы после экспорта в SVG все нужные объекты **надёжно распознались** парсером и попали в редактор сайта.

Связанный технический документ: [design-template-importer.md](./design-template-importer.md).

---

## 1. Что сдаём

| Формат | Когда |
|---|---|
| **SVG** | Один размер, одна или несколько страниц (один файл — одна страница) |
| **ZIP со SVG** | Несколько страниц одного размера (файлы в корне: `page-1.svg`, `page-2.svg`…) |
| **ZIP с папками размеров** | Несколько физических размеров одной семьи: `90x50/page-1.svg`, `50x90/page-1.svg` |

Master-исходник (`.cdr` / `.ai`) храните у себя. В CRM уходит только SVG/ZIP.

### Размер страницы

- В корневом SVG обязательно укажите размер в миллиметрах, например:  
  `width="90mm"` `height="50mm"` (и согласованный `viewBox`).
- Имя папки в multi-size ZIP (`90x50`) — подсказка; **истина — мм в SVG**.

## 2. Как задавать имена (самое важное)

Имя объекта/группы читается из **Object Data / ID** (в SVG: `id`, иногда `inkscape:label` / `data-name`).

Пишите **латиницей**, префиксы строго как ниже. Регистр лучше нижний: `text_title`, не `Text_Title`.

| Префикс / имя | Что это | Что должно быть в файле |
|---|---|---|
| `locked_bg` | Заблокированный фон страницы | Группа или объекты фона. Без этого слоя фон в редакторе **не создаётся**. PNG/JPG внутри `locked_bg` остаются в фоне. |
| `photo_` / `photo_cover` / `photo_1` | Поле под фото клиента | Только **прямоугольник** (`rect`). Можно группу `photo_*` с rect внутри. |
| `text_` / `text_title` / `text_phone` | Редактируемый текст | Живой **текст**, не кривые. |
| `text_title_center` | Текст + выравнивание | См. §3 |
| `decor_` / `decor_shape` | Декор, который можно двигать в редакторе | `rect` / `circle` / `ellipse` / `path` / `polygon` / растровый `image` |
| *(без имени)* | Авто-декор | Те же фигуры → `decor_auto_*`. Подложка на весь лист без имени обычно **не** становится декором. |
| `trim`, `bleed`, `safe` | Направляющие печати | Прямоугольники зон |
| `hidden_*`, `guide_*` | Техническое | В шаблон клиента не попадает |

### Автонумерация

Можно назвать просто `photo_`, `text_`, `locked_bg` — парсер сделает `photo_1`, `text_1`, `locked_bg_1`.  
Осмысленные имена (`text_title`, `photo_avatar`) предпочтительнее для ТЗ и правок.

## 3. Текст: выравнивание и строки

### Выравнивание через имя (рекомендуем)

| Имя в Corel | Id в системе | Выравнивание в редакторе |
|---|---|---|
| `text_title_center` | `text_title` | по центру |
| `text_name_right` | `text_name` | по правому краю |
| `text_phone_left` | `text_phone` | по левому краю |
| `text_center` | `text_1` (авто) | по центру |

Суффиксы: `_center` / `_right` / `_left` (также `_middle` / `_end` / `_start`).  
В id суффикс **не остаётся** — только свойство `textAlign`, как кнопка «по центру» в редакторе.

### Если суффикса нет

Парсер смотрит `text-anchor` / `text-align` в SVG и при склейке строк может **угадать** center/right по координатам. Для критичных блоков лучше явный суффикс `_center` / `_right`.

### Несколько строк = одно поле

Чтобы «Имя» и «Должность» были **одним** текстовым полем:

1. Одно имя на все строки: `text_name` или `text_name_center`.
2. Либо несколько `<text>` / `tspan` с **одинаковым** id / в одной группе `text_*`.

Разные id (`text_name` и `text_job`) → **два** отдельных поля.

### Обязательно

- Текст = текст, **не** «в кривые».
- Шрифты: либо стандартные/уже в CRM, либо приложите файлы шрифтов в ZIP импорта.
- Цвет — через `fill` (в т.ч. стили Corel `.fil0 { fill:#… }`).
- Поворот (вертикальный текст) — через transform группы/текста, имя всё равно `text_*`.

---

## 4. Фото-поля

1. Прямоугольник нужного размера.
2. Имя `photo_` / `photo_1` / `photo_cover` (на rect или на группе с rect внутри).
3. Не превращайте рамку фото в path/кривую — парсер ждёт **rect**.

В редакторе пустое поле покажет плейсхолдер (рамка + иконка камеры).

---

## 5. Фон и декор

### Фон

1. Соберите всю «нередактируемую» графику в слой/группу **`locked_bg`**.
2. Туда же — фоновые PNG/JPG, если они не должны быть отдельным декором.
3. Без `locked_bg` интерактивные объекты будут, но **фонового слоя страницы не будет**.

### Декор (двигать можно)

- Явно: `decor_wave`, `decor_logo`.
- Или без имени → `decor_auto_*` (path, rect, image…).
- Градиенты Corel (`fill:url(#…)`) поддерживаются.
- Растр PNG/JPG как отдельный объект → `decor_*` или авто; внутри `locked_bg` → фон.

---

## 6. Направляющие (по желанию)

Прямоугольники с именами:

- `trim` — обрез
- `bleed` — вылет
- `safe` — безопасная зона

По ним считаются подсказки prepress в шаблоне. Сверяйте с производством.

---

## 7. Чеклист перед сдачей

- [ ] Размер SVG в **мм** совпадает с нужным форматом продукта  
- [ ] Есть слой **`locked_bg`** на весь фон  
- [ ] Все фото-поля — **rect** с префиксом **`photo_`**  
- [ ] Весь клиентский текст — живой **text** с префиксом **`text_`** (не кривые)  
- [ ] Центр/право — суффикс **`_center` / `_right`** в имени  
- [ ] Строки одного блока текста — **одно имя**  
- [ ] Декор для редактора — `decor_*` или осознанно без имени  
- [ ] Шрифты доступны (в CRM или в ZIP)  
- [ ] Экспорт SVG: текст не в кривые; имена объектов сохранились в id  

---

## 8. Частые ошибки

| Ошибка | Что будет |
|---|---|
| Текст переведён в кривые | Не станет `text_*`, останется в фоне/декоре |
| Нет `locked_bg` | Нет фонового слоя в редакторе |
| Фото — path, не rect | Поле фото не создастся |
| «НАЗВАНИЕ» и «ОРГАНИЗАЦИИ» с разными id | Два поля вместо одного |
| Нужен центр, имя просто `text_title` | Может уехать влево, если геометрия не угадается → лучше `text_title_center` |
| Фон без имени на весь лист | Обычно не станет лишним декором (ок), но лучше явно `locked_bg` |

---

## 9. Мини-пример структуры слоёв (визитка)

```
locked_bg          ← весь фон, графика, фоновые картинки
photo_avatar       ← rect под фото
text_title_center  ← «НАЗВАНИЕ» + «ОРГАНИЗАЦИИ» (одно имя, две строки)
text_phone_right   ← телефон / контакты справа
decor_ornament     ← опционально, если элемент должен двигаться
trim / bleed / safe ← по необходимости
```

После импорта в CRM:

- `text_title` с выравниванием **center**
- `text_phone` с выравниванием **right**
- `photo_avatar`, фон `locked_bg`

---

## 10. Куда сдавать

Готовый **SVG** или **ZIP** — в админку CRM → каталог шаблонов → импорт.  
При правках макета — снова экспорт SVG и **reimport** / повторный импорт (старые шаблоны сами не обновляются).
